## 4. 성능 튜닝

### (1) TCP 버퍼 크기

```bash
# 현재 설정 확인
cat /proc/sys/net/ipv4/tcp_rmem   # 읽기 버퍼
cat /proc/sys/net/ipv4/tcp_wmem   # 쓰기 버퍼

# 읽기: min 4KB, default 87KB, max 6MB
# 쓰기: min 4KB, default 16KB, max 4MB

# 튜닝 (높은 처리량)
sysctl -w net.ipv4.tcp_rmem="4096 131072 67108864"
sysctl -w net.ipv4.tcp_wmem="4096 65536 67108864"
```

### (2) TCP Window Scaling

```bash
# 초기 윈도우: 64KB (16비트로 제한)
# Scaling: 1Mbps 링크에서 너무 작음

# 활성화
sysctl -w net.ipv4.tcp_window_scaling=1

# 효과: 최대 1GB까지 가능
```

### (3) TCP Fast Open (TFO)

```bash
# 3-way handshake 생략 (초기 데이터 함께)
sysctl -w net.ipv4.tcp_fastopen=3

# 효과: 연결 시간 33% 단축
```

## 5. Wireshark 분석

### 실전 필터

```

tcp.port == 8080

// 느린 응답 (200ms+)
tcp.time_delta > 0.2

// 재전송 감지
tcp.analysis.retransmission

// 세그먼트 오류
tcp.analysis.lost_segment
```
