## 1. AMM이란?

### 기존 거래소 vs AMM

```
기존 거래소 (Centralized Exchange, CEX):
├─ Order Book: 사용자들의 호가 모음
├─ Bid/Ask 매칭
├─ 중간자: 거래소 운영자
├─ 예: Coinbase, Binance

AMM (Decentralized Exchange, DEX):
├─ Pool: 토큰 쌍의 자동 자금 풀
├─ 공식: 수학적 곡선 (x*y=k)
├─ 중간자: 불필요
├─ 예: Uniswap, Curve, Balancer
```

### 상수곡선 (x*y=k)

```
Uniswap V2 공식:

Pool: Token A + Token B

사용자가 Token A Δx를 입금하면,
Token B Δy를 받음

수학:
(x + Δx) * (y - Δy) = x*y

정렬:
Δy = y - [x*y / (x + Δx)]

예: ETH/USDC 풀
├─ Reserve A: 100 ETH
├─ Reserve B: 300,000 USDC
├─ k = 100 × 300,000 = 30,000,000

사용자가 10 ETH를 입금:
├─ 새로운 x = 100 + 10 = 110
├─ 새로운 y = 30,000,000 / 110 = 272,727
├─ Δy = 300,000 - 272,727 = 27,273 USDC
└─ 실제 가격: 27,273 / 10 = 2,727 USDC/ETH (시장가 3,000보다 낮음)
   → 슬리피지: 2,727 / 3,000 = 90.9% (엄청난 손실!)

원인: 풀 크기가 작음 (100 ETH)
```

## 2. Uniswap V3: 집중된 유동성

### V2 vs V3

```
V2 (고정 범위):
┌──────────────────────────────┐
│ ├─ Price: $2,000 ~ $4,000    │
│ ├─ Liquidity 균등 분배         │
│ └─ 범위 밖 = 손실 (Fee 못 받음) │
└──────────────────────────────┘

V3 (범위 설정 가능):
┌──────────┬──────────┬──────────┐
│ $2,000~  │ $2,500~  │ $3,000~  │
│ $2,500   │ $3,000   │ $3,500   │
└──────────┴──────────┴──────────┘

LP가 선택:
├─ 범위 $2,500-$3,500: 수익성 높음 (fee 자주)
└─ 범위 $1,000-$5,000: 위험 낮음 (범위 밖 가능성 낮음)
```

### 공식

```
V3 유동성 = L (virtual liquidity)

전체 token 계산:
├─ Amount A = L * (√Pb - √Pa)
└─ Amount B = L * (1/√Pa - 1/√Pb)

예: ETH/USDC
├─ Pa = $2,000, Pb = $4,000
├─ L = 1,000,000
├─ Amount ETH = 1,000,000 × (√4000 - √2000) = 1,000,000 × 21.2 = 21,200,000
   (그런데 이건 virtual balance 기준)

실제로는 Pool 계산이 더 복잡함
```

## 3. Curve Finance: 안정 자산 전문

### Curve 공식

```
Curve는 안정 자산(USDC, USDT, DAI)의 교환에 최적화

공식: x^3*y + x*y^3 = k

특징:
├─ 현물 가격 근처: 거의 선형 (슬리피지 적음)
├─ 극단적 가격: 급격한 곡선 (극도의 슬리피지)
└─ 예: USDC ↔ USDT (거의 1:1, 슬리피지 0.01%)
```

### Curve의 우수성

```
Uniswap V3 (ETH/DAI) vs Curve (DAI/USDC)

$1,000,000 거래:

Uniswap:
├─ 슬리피지: 5% (시장가보다 5% 높음)
├─ 손실: $50,000
└─ Fee: 0.3% = $3,000

Curve (안정 자산):
├─ 슬리피지: 0.01%
├─ 손실: $100
└─ Fee: 0.04% = $400
```

## 4. 유동성 제공자 (LP) 경제학

### LP 수익

```
LP가 Pool에 1 ETH + 3,000 USDC 입금

수익 1: 거래 수수료
├─ 풀에서 발생한 거래의 0.3% (Uniswap V2)
├─ 월 10,000 USDC 거래 → 30 USDC 수익
└─ APY: 120% (년 3,600 USDC)

수익 2: 인센티브 (유동성 채굴)
├─ 프로토콜이 제공하는 토큰 (예: UNI)
├─ 추가 APY: 200%+ (초기 단계)
└─ 2년 후 감소

손실: 무상 손실 (Impermanent Loss, IL)
```

### 무상 손실 (IL)

```
시나리오: ETH 가격이 $3,000 → $6,000으로 상승

LP 전략:
├─ Option A: Hold (그냥 보유)
│  ├─ 1 ETH × $6,000 = $6,000
│  └─ 3,000 USDC = $3,000
│  └─ 총: $9,000
│
└─ Option B: LP (Uniswap 제공)
   ├─ 가격이 올라서 LP는 ETH를 팔아서 USDC 사기
   ├─ 최종 보유: 0.707 ETH + 4,242 USDC
   ├─ 계산: 0.707 × $6,000 + $4,242 = $8,484
   └─ 손실: $9,000 - $8,484 = $516

IL % = ($8,484 - $9,000) / $9,000 = -5.7%

극단적 가격 변동:
├─ 2배 변동: ~5.7% IL
├─ 4배 변동: ~20.5% IL
└─ 10배 변동: ~47.5% IL

공식:
IL(x) = 2√x / (1+x) - 1
(여기서 x = 가격 비율)
```

### IL 회피 전략

```
1. 낮은 변동성 쌍 선택
   ├─ DAI/USDC (IL < 0.5%)
   └─ 안정 자산 풀

2. 좁은 범위 (V3)
   ├─ $2,500-$3,000: IL 낮음, Fee 높음
   └─ 활발한 모니터링 필요

3. 집중된 범위로 높은 APY 추구
   ├─ Fee로 IL 보충
   └─ 장기 수익 계산
```

## 5. 플래시 론 (Flash Loan)

### 개념

```
플래시 론 = 담보 없이 빌린 뒤 같은 블록 내 반환

규칙:
├─ 1. 빌리기: Pool에서 원하는 양 빌림
├─ 2. 실행: 그 돈으로 뭐든 할 수 있음
├─ 3. 반환: 원래 양 + 수수료(0.09%) 반환
└─ 4. 체크: 모두 같은 트랜잭션 내

비용:
├─ 반환 안 하면? Tx 전체 revert (스마트 컨트랙트 이용)
└─ 수수료: 0.09% (매우 저렴)
```

### 플래시 론 사례

```
Uniswap V2 Flash Loan 100,000 USDC

Solidity:
function executeFlashLoan(uint amount) public {
    // 1. 플래시 론 요청
    flashLoan(amount);
}

function uniswapV2Call(address sender, uint amount0, uint amount1, bytes calldata data) external {
    // 2. 여기서 돈을 가짐!
    uint amountDue = amount + amount * 9 / 10000;  // 0.09% 수수료
    
    // 3. 다른 곳에서 더 싼 가격으로 USDC 구매
    //    또는 차익거래 실행
    uint profit = doArbitrage(amount);
    
    // 4. 반환 (꼭 필요!)
    USDC.transfer(address(pair), amountDue);
}
```

### 플래시 론 공격

```
2020년 bZx 해킹:

1. 플래시 론으로 7,500,000 USDC 빌림
2. 대출 프로토콜의 가격 오라클 조작
3. 담보 가치 폭락, 대출 상환 받음
4. 플래시 론 반환
5. 이익: ~350,000 달러 (비용 67달러)

해결책:
├─ 오라클 다중화
├─ 타임락 도입
└─ 가격 변경 제한
```

## 6. 차익거래 (Arbitrage)

### 크로스 거래소 차익

```
상황:
├─ Uniswap: ETH = 3,000 USDC
└─ Curve: ETH = 3,100 USDC (가격 다름!)

차익거래:
1. Uniswap에서 10 ETH를 3,000 USDC/ETH 가격으로 구매
   └─ 비용: 30,000 USDC
2. Curve에서 10 ETH를 3,100 USDC/ETH 가격으로 판매
   └─ 수익: 31,000 USDC
3. 수익: 31,000 - 30,000 = 1,000 USDC (가스비 제외)

이 과정이 모두 자동 (Bot)으로 진행
→ 가격 수렴
```

### 메매봇 (MEV: Maximal Extractable Value)

```
Flashbots Protect:
├─ 트랜잭션을 Private Pool로 보냄
├─ 메매봇 (Sandwich Attack) 방지
└─ 대신 더 높은 Fee 지불

Sandwich Attack 예시:
1. 사용자의 거래 감지
2. 사용자 거래 전에 같은 거래 먼저 실행
3. 가격 변동
4. 사용자의 거래 실행 (더 나쁜 가격)
5. 공격자 거래 역거래로 수익 확보
6. 손실: 사용자 100-500 달러
```
